// 01 — Overview

What it does

This bash script automates the full process of installing NI Circuit Design Suite 14.0 on Linux. It handles Wine installation, a dedicated 32-bit Wine prefix, dependency setup, and the Multisim installer — all in a single run across all supported distros.

🍷
Wine Management
Automatically installs the right Wine version for your distro and configures a dedicated 32-bit prefix at ~/.multisim32 — fully isolated from your existing Wine setup.
⚙️
Dependency Setup
Installs corefonts, mdac27, and jet40 via winetricks — the exact stack Multisim 14.0 needs to run reliably.
📦
Auto Download & Install
Fetches the official NI Circuit Design Suite installer directly from National Instruments' servers and runs it through Wine for you.
🖥️
Desktop Integration
Patches the .desktop launcher on Debian and Fedora systems so Multisim appears correctly in your application menu with the right Wine prefix.
Why version 14.0?

Multisim 14.0 is the newest version that works reliably on Linux with minimal issues, while still including most features available in 14.3. Want to know more about the latest version? Check out this blog post ↗

Usage

Prerequisites: wgetunzip

1
Clone the repository
bash
git clone --depth 1 https://github.com/ghepardoman/NI-Multisim-14-for-Linux.git
cd NI-Multisim-14-for-Linux
2
Make the script executable
chmod +x install.sh
3
Run the installer
./install.sh
⚠️ Do not run as root. The script uses sudo internally where needed.
// 04 — Internals

How it works

01
Distro Detection
Reads /etc/os-release to identify your distribution family and selects the correct installation path.
02
Conflict Removal
Checks for and removes any existing Wine installations that may conflict with the version required by Multisim.
03
Wine Install
Installs Wine from the appropriate source: Chaotic AUR / yay on Arch, apt on Debian/Ubuntu, dnf on Fedora, zypper on openSUSE.
04
Wine Prefix
Creates a dedicated 32-bit Wine prefix at ~/.multisim32 with Windows XP compatibility mode — essential for Multisim to function.
05
Dependencies
Installs corefonts, mdac27, and jet40 via winetricks for full compatibility.
06
Download & Cleanup
Downloads the official NI installer, runs it via Wine, then removes the ZIP and extracted directory automatically.
Wine prefix setup command
# Isolated 32-bit prefix with Windows XP compatibility
WINEARCH=win32 WINEPREFIX="$HOME/.multisim32" winecfg -v winxp
// 05 — Known Issues

Notes & Known Issues

  • Arch Linux — users are prompted whether to use Chaotic AUR (fast, pre-built) or compile from AUR (slow). Chaotic AUR is strongly recommended.
  • Arch Linux — a package starting with "wine" may be wrongly queried as "wine" when checking for conflicts. If pacman fails, remove that package manually before re-running the script.
  • openSUSE — Wine continuously tries to open winedbg, halting operations. The included forceClosewinedbg.sh script auto-kills it. After reboot, just close the winedbg window — Multisim will run fine.
  • The Wine prefix is stored at ~/.multisim32 and is completely separate from any existing Wine setup you may have.
  • A system reboot is recommended after installation.
  • If Multisim doesn't appear in your app launcher after install, run:
    update-desktop-database ~/.local/share/applications
